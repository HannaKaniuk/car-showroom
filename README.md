# Car Showroom

Віртуальний автосалон — односторінковий застосунок на React з каталогом автомобілів, фільтрацією та системою відгуків.

🔗 **Live demo:** https://hannakaniuk.github.io/car-showroom/

## Опис проєкту

Застосунок завантажує дані про автомобілі з [DummyJSON API](https://dummyjson.com) (категорія `vehicle`) і дозволяє переглядати каталог, фільтрувати результати та залишати відгуки. Локальні коментарі зберігаються в браузері й не зникають після перезавантаження сторінки.

## Функціонал

### Каталог (`/`)

- завантаження списку автомобілів з API
- пошук за назвою, брендом і описом
- фільтр за брендом
- сортування: назва, ціна, рейтинг
- валідація довжини пошукового запиту (макс. 100 символів)
- лічильник знайдених результатів

### Сторінка автомобіля (`/vehicles/:id`)

- детальна інформація: ціна, рейтинг, наявність, гарантія, доставка
- список відгуків з API
- форма додавання коментаря з валідацією
- збереження нових відгуків у `localStorage`

### Додатково

- сторінка 404 для неіснуючих маршрутів
- стани завантаження та помилок з кнопкою повторного запиту
- адаптивна верстка (420–1440px)
- доступність: ARIA-атрибути, семантична розмітка, клавіатурна навігація

## Локальний запуск

**Вимоги:** Node.js 18+

```bash
# клонування репозиторію
git clone https://github.com/HannaKaniuk/car-showroom.git
cd car-showroom

# встановлення залежностей
npm install

# запуск dev-сервера
npm run dev
```

Застосунок буде доступний за адресою [http://localhost:5173](http://localhost:5173).

### Інші команди

```bash
npm run build    # production-збірка
npm run preview  # перегляд збірки локально
npm run lint     # перевірка коду ESLint
```

## Структура проєкту

```
src/
├── api/           # HTTP-запити до DummyJSON
├── components/    # UI-компоненти (картки, фільтри, форми)
├── pages/         # сторінки маршрутів
├── types/         # TypeScript-інтерфейси
└── utils/         # валідація, форматування, робота з localStorage
```

## Деплой

Застосунок розгорнуто на **GitHub Pages**: https://hannakaniuk.github.io/car-showroom/

Повторний деплой після змін:

```bash
./scripts/deploy-pages.sh
```

| Параметр | Значення |
|----------|----------|
| Build command | `VITE_BASE_PATH=/car-showroom/ npm run build` |
| Output directory | `dist` |
| Base path | `/car-showroom/` |

Також підтримується розгортання на [Vercel](https://vercel.com) або [Netlify](https://netlify.com) — для SPA-маршрутизації налаштовано `vercel.json`, `netlify.toml` та `public/_redirects`.
